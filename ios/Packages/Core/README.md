# Core

Foundation package: DI facade (Factory), logging, the `DomainError` taxonomy, result types, date/locale utilities. Everything else depends on this; it depends on nothing else in-repo. See [docs/planning/02-ios-architecture.md](../../../docs/planning/02-ios-architecture.md) and [ADR-0012](../../../docs/adr/0012-dependency-injection.md).

**Status:** implemented and tested (`swift test` — 6/6 passing). Depends only on Foundation + Factory (the one third-party dependency this module is allowed, isolated behind `@_exported import Factory` so every other module resolves DI via `import Core`, never `import Factory` directly — ADR-0012). Ships `DomainError`/`FieldError`, `AppLogger`/`OSAppLogger` (one `os.Logger` per category, lazily created), and `AppLocale` (RTL-aware locale wrapper consumed by `DomainKit.AppConfiguration`). `DIRegistrationPoints.swift` declares the one Factory `Container` extension that has a real default here (`appLogger: Factory<AppLogger> { OSAppLogger.shared }`) rather than a `fatalError` placeholder — every other cross-module registration point (repositories/use cases) lives in `DomainKit` instead, since it's the lowest layer visible from every Feature package; see `ios/Packages/DomainKit/README.md`.
