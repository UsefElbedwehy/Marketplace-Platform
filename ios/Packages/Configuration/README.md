# Configuration

Client-side Configuration Engine: loads the embedded build-time schema, fetches the runtime config bundle (`GET /v1/config`), caches it in SwiftData, exposes typed accessors (`AppConfig`, `FeatureFlags`, `LocaleConfig`, `CurrencyConfig`) — no hardcoded currency/country/business constants anywhere. See [docs/planning/07-configuration-whitelabel-theme.md §2](../../../docs/planning/07-configuration-whitelabel-theme.md#2-configuration-engine-client-side) and [ADR-0004](../../../docs/adr/0004-configuration-engine.md).

**Status:** implemented and tested (`swift test` — 11/11 passing). Depends on Core + Networking. Ships `ConfigDTO`/`ThemeDTO` (field-for-field mirrors of `configs/clients/<id>/{config,theme}.json`), a bundled build-time default (`BundledDefaults`, generated from the same `configs/clients/default/*.json` the backend seeds from — see `Resources/`), a SwiftData cache (`CachedConfigRecord`/`CachedThemeRecord`, keyed by an explicit `clientId` param rather than the payload's own field, storing the raw decoded-JSON payload + ETag + fetch time — ADR-0013), and `ConfigurationDataSource`'s cache-then-network policy (cache hit + 304 → cached; fresh 200 → cache + return; network failure with no cache → bundled default). `DataKit`'s `ConfigRepositoryImpl` maps `ConfigDTO` → `DomainKit.AppConfiguration`; `DesignSystem`'s `ThemeStore` consumes `ThemeDTO` directly (theme has no `DomainKit` repository — see that file's doc comment).
