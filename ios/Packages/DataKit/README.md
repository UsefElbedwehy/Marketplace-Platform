# DataKit

Repository implementations fulfilling `DomainKit` protocols: DTOs, mappers, cache policy (SwiftData-backed, network-first or cache-then-network per data type). See [docs/planning/02-ios-architecture.md §5](../../../docs/planning/02-ios-architecture.md#5-local-persistence--swiftdata-as-a-cache-not-a-source-of-truth) and [ADR-0013](../../../docs/adr/0013-local-persistence.md).

**Status:** implemented and tested (`swift test` — 18/18 passing). Depends on DomainKit + Networking + Configuration. Ships `ConfigRepositoryImpl` (maps `Configuration.ConfigDTO` → `DomainKit.AppConfiguration`) and `AuthRepositoryImpl` (mints a session via `Networking.DevAuthEndpoint`, persists via `Networking.TokenStore`, maps `Networking.StoredSession` ↔ `DomainKit.AuthSession` — the only place those two types meet). No theme repository here; see `DesignSystem/Theme/ThemeStore.swift`'s doc comment for why theme skips the Domain/Data layers entirely.

**Extended in Phase 4** with `CategoryRepositoryImpl` and `ListingRepositoryImpl` — the only places `CatalogDTO`/`ListingDTO` translate to/from `DomainKit`'s catalog/listing entities. `DTOs/JSONScalar.swift` is a minimal internal (non-public) `Codable` enum (`string`/`number`/`bool`/`array`/`null`) used to decode the contract's heterogeneous JSON blobs (`attributesIndex`, `validation`, `condition.equals`) without building a full recursive JSON-value type. `ListingRepositoryImpl.encodeFilters(_:)` JSON-encodes equality/range attribute filters for the `filters` query param; `requiresAuth` is `filters.mine || filters.status != nil` — anything beyond the public "published" default needs a bearer token.
