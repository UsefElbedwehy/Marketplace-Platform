# DesignSystem

The Theme Engine + reusable UI Core (buttons, cards, inputs, search, dropdowns, bottom sheets, dialogs, snackbars, loading/shimmer/skeleton, gallery, price view, listing/seller/review cards, chat bubble, notification card, empty/error/offline states). **The only place raw color/font literals are allowed** — enforced by `tooling/lint/boundaries.sh`. See [docs/planning/07 §4](../../../docs/planning/07-configuration-whitelabel-theme.md#4-theme-engine) and [ADR-0005](../../../docs/adr/0005-theme-engine.md).

**Status:** implemented and tested — 9 logic tests (`swift test`) + 3 real snapshot tests (`xcodebuild test` against an iOS Simulator, using `swift-snapshot-testing`; plain `swift test` builds for macOS and would render through AppKit, not iOS, so the snapshot suite is `#if canImport(UIKit)`-guarded to only run on the real target platform). Depends on Core + Configuration (tokens only — never DomainKit, per 01-system-architecture.md §4).

Ships: `Theme`/`SemanticColors`/`Typography`/`ThemeShape` (resolved from `Configuration.ThemeDTO`, hex-parsed via `Color(hex:)`), `ThemedRoot` (injects `\.theme` + a color-scheme-reactive `\.semanticColors`, plus `.tint()` at the true root so system chrome picks it up), `ThemeStore` (`@Observable @MainActor`, wraps `ConfigurationDataSource.loadTheme()`), and a first representative component slice: `PrimaryButton`/`SecondaryButton`, `Card`, `Badge`, `AppTextField`, `LoadingIndicator`/`ShimmerView`, `EmptyStateView`/`ErrorStateView`/`OfflineBanner`. Remaining components (search, sheets, dialogs, snackbars, listing/seller/review cards, chat bubble) follow the same token-driven pattern as Features that need them land in Phase 4/6.

**Known cosmetic finding:** on iOS 26's new "Liquid Glass" floating tab bar, the selected tab's glyph/label color did not pick up `.tint()` in manual verification (every other themed element — buttons, cards — renders the exact theme color correctly, confirmed via snapshot tests). Not chased further with a static `UITabBar.appearance()` override since that would conflict with the live theme-hot-swap guarantee (ADR-0005); revisit if it persists on GA iOS 26.
